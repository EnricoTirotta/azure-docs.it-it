# Inserimento dell'applicazione nella raccolta di applicazioni Azure Active Directory

Prima di poter inserire un'applicazione che supporta Single Sign-On con Azure Active Directory nella [raccolta di Azure AD](http://azure.microsoft.com/marketplace/active-directory/all/) e in Azure Marketplace, il team di Azure AD deve verificare l'integrazione. A questo scopo, è necessario inviare le informazioni seguenti all'indirizzo <waadpartners@microsoft.com>:

- Nome utente e password di un account utente di prova in un'istanza di prova di Azure Active Directory in grado di accedere all'applicazione
- URL e/o istruzioni su come questo utente di prova deve accedere all'applicazione
- Fornire una breve descrizione di come i clienti creano la connessione tra l'applicazione e Azure AD. Alcuni esempi:
  - Connessione integrata nell'applicazione usando il framework di consenso di Azure AD
  - Configurando Azure AD come provider di identità (SAML 2.0, WS-Federation o OpenID Connect) nel portale di amministrazione dell'applicazione
  - Richiedendo a qualcuno dell'organizzazione di creare manualmente la connessione
- Se l'applicazione supporta la configurazione di Azure AD come provider di identità attraverso un portale di amministrazione self-service, specificare anche un account amministratore di prova per la verifica.

> L'applicazione è stata integrata in Azure AD e testata (vedere [Scenari di autenticazione di Azure Active Directory](active-directory-authentication-scenarios.md))?
>
> ID client dell'applicazione, indicato nei dettagli dell'applicazione all'indirizzo https://manage.windowsazure.com:

# Passaggi successivi per l'uso di Azure AD

## Gestire gli utenti in Azure AD
La gestione degli account utente non si limita alla creazione o all'eliminazione degli utenti e all'assegnazione delle autorizzazioni. È possibile generare report sugli accessi e sull'utilizzo, impostare criteri password, abilitare la reimpostazione delle password self-service, configurare l'autenticazione a più fattori e altro ancora. [Altre informazioni sulla gestione degli utenti](active-directory-create-users.md)

## Sincronizzare e gestire le password
Sincronizzare le password utente in modo che gli utenti possano accedere ai servizi cloud Microsoft, ad esempio Office 365, Dynamics CRM e Microsoft Intune, con la stessa password che usano per accedere alla rete locale. [Altre informazioni sulla gestione delle password](active-directory-manage-passwords.md)

## Gestire gruppi in Azure AD
Usare i gruppi per assegnare un set di autorizzazioni e diritti comune a molti account alla volta, anziché singolarmente per ogni account. È anche possibile sincronizzare i gruppi da Windows Server Active Directory locale. Altre informazioni su come gestire i gruppi in Azure Active Directory [Altre informazioni sulla gestione dei gruppi](active-directory-manage-groups.md)

## Imporre criteri di accesso condizionale per i dispositivi mobili registrati
Registrazione dispositivo di Azure AD consente di eseguire il provisioning dei dispositivi con un'identità usata per autenticare il dispositivo quando l'utente esegue l'accesso. Il dispositivo autenticato e gli attributi del dispositivo possono quindi essere usati per imporre criteri di accesso condizionale per le applicazioni locali e ospitate nel cloud. [Altre informazioni su Registrazione dispositivo di Azure AD](active-directory-conditional-access.md).

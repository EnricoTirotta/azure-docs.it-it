# Panoramica del database SQL

Il database SQL è un servizio di database relazionale nel cloud basato sul motore di Microsoft SQL Server e con funzionalità di importanza strategica incorporate. Il database SQL offre prestazioni prevedibili, scalabilità, continuità aziendale, protezione dei dati con esigenze di amministrazione quasi nulle per gli sviluppatori di cloud e Solution Architect. Aspetto ancora più importante: il database SQL supporta gli strumenti, le librerie e le API del sistema SQL Server esistente, semplificando la migrazione al cloud. È possibile rendere un database SQL [operativo in pochi secondi](sql-database-get-started.md), senza dover gestire macchine virtuali o un'infrastruttura. La funzionalità di tipo DBA a livello di programmazione consente DevOps efficienti sviluppo e la gestione automatica garantisce una manutenzione pressoché nulla dei database. Questo consente allo sviluppatore di concentrarsi sullo sviluppo e la gestione dell'applicazione.

Il database SQL offre una [scalabilità verticale](sql-database-service-tiers.md) o [orizzontale](sql-database-elastic-scale-get-started.md) per migliaia di database, con prestazioni prevedibili che possono essere aumentate o ridotte, in base alle necessità. Un [contratto di servizio](http://azure.microsoft.com/support/legal/sla/) leader del settore con livelli di disponibilità pari al 99,99% assicura tempi di inattività pressoché nulli per le applicazioni. La protezione dei dati incorporata tramite controllo, ripristino e replica geografica garantisce la protezione costante dei dati. Il database SQL consente di ripristinare i dati in modo temporizzato da qualsiasi transazione effettuata entro un periodo massimo di 35 giorni. È possibile inoltre replicare i dati in un'area Azure di propria scelta e implementare in modo semplice un criterio di ripristino di emergenza geografico adatto a specifiche esigenze aziendali. La [versione più recente del database SQL](sql-database-preview-whats-new.md) (V12) offre una compatibilità quasi completa con il motore di SQL Server.

Il database SQL è disponibile nei [livelli di servizio](sql-database-service-tiers.md) Basic, Standard e Premium che supportano carichi di lavoro semplici fino a carichi di lavoro complessi. È pertanto possibile passare da un livello a un altro o combinarne diversi per progettazioni di applicazioni innovative. Con la potenza e la portata di Azure, è possibile combinare e integrare i servizi di Azure con il database SQL per soddisfare le esigenze esclusive di progettazione di app moderne dell'azienda, promuovere l'efficienza in termini di costi e di risorse e sfruttare nuove opportunità commerciali.

È possibile iniziare con una [versione di valutazione gratuita](http://azure.microsoft.com/pricing/free-trial/) per scoprire [come creare il primo database SQL](sql-database-get-started.md).

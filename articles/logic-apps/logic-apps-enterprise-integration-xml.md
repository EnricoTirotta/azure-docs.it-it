# <a name="xml-processing"></a>Elaborazione XML
È possibile convalidare ed elaborare i documenti XML che si scambiano con i partner negli scenari B2B più facilmente quando si usa Enterprise Integration Pack con le app per la logica di Azure. Ecco i modi in cui è possibile elaborare i messaggi XML:

* [Convalida dei messaggi XML](logic-apps-enterprise-integration-xml-validation.md "Informazioni sulla convalida dei messaggi XML"): consente di convalidare un messaggio proveniente da un endpoint di origine rispetto a uno schema specifico.
* [Trasformazione dei messaggi XML](../logic-apps/logic-apps-enterprise-integration-transform.md "Informazioni sulle mappe e le trasformazioni di messaggi XML"): consente di convertire un messaggio XML in base ai requisiti di un endpoint di destinazione.
* [Codifica e decodifica del file flat](logic-apps-enterprise-integration-flatfile.md "Informazioni sulla codifica o decodifica del file flat"): consente di codificare o decodificare un file flat. 
* [XPath](https://msdn.microsoft.com/library/mt643789.aspx): consente di arricchire un messaggio ed estrarre le proprietà specifiche del messaggio. È possibile usare le proprietà estratte per indirizzare il messaggio a un endpoint intermedio o di destinazione.

## <a name="try-out-xml-processing"></a>Provare l'elaborazione XML
[Distribuire un'app per la logica completamente operativa](https://github.com/Azure/azure-quickstart-templates/tree/master/201-logic-app-veter-pipeline) usando le funzionalità XML di App per la logica di Azure.
